# Prompt para Gamma — Presentación Trabajo de Grado

Usa este prompt en Gamma para generar una presentación profesional, clara y académica.

---

## Prompt

Crea una presentación profesional en español para la defensa de un trabajo de grado de Ingeniería en Informática.

### Tema
Desarrollo de un sistema web llamado **COPS Platform** para la gestión integral de tickets, servicios técnicos, proyectos, pagos a técnicos y reportes operativos para una empresa de servicios electrónicos llamada **COPS Electronics, C.A.**

### Objetivo de la presentación
Explicar el problema, la solución, la arquitectura, el funcionamiento del sistema y el valor técnico/empresarial del proyecto.

### Audiencia
Jurado universitario con perfil técnico, pero la presentación debe ser comprensible, elegante y muy bien estructurada.

### Estilo visual
- profesional
- tecnológico
- limpio
- moderno
- colores azul, blanco y gris
- iconografía sutil
- diagramas claros
- poco texto por diapositiva
- enfoque ejecutivo + técnico

### Tono
- académico
- seguro
- claro
- convincente
- técnico pero explicable

### Estructura solicitada

1. **Portada**
   - Título del proyecto
   - Nombre del autor
   - Carrera: Ingeniería en Informática
   - Universidad

2. **Planteamiento del problema**
   - dificultades en la gestión manual de servicios técnicos
   - falta de trazabilidad
   - problemas de coordinación y control

3. **Objetivo general y objetivos específicos**

4. **Descripción de la solución propuesta**
   - qué es COPS Platform
   - qué módulos incluye

5. **Arquitectura del sistema**
   - Next.js App Router
   - TypeScript
   - Firebase Authentication
   - Firestore
   - Cloudinary/Firebase Storage para archivos
   - Server Actions
   - modo local/demo y modo firebase

6. **Módulos principales**
   - Dashboard
   - Tickets
   - Pipeline (tablero y calendario)
   - Pagos
   - Reportes
   - Usuarios

7. **Flujo operativo del ticket**
   - creación
   - asignación
   - llegada al sitio
   - ejecución
   - pausa/reprogramación
   - finalización
   - comprobante
   - pago

8. **Control de acceso y roles**
   - técnico
   - coordinador
   - gerente
   - vicepresidente
   - presidente

9. **Reportes y toma de decisiones**
   - KPIs
   - reportes por cliente
   - reportes por técnico
   - caso Bancaribe: cupones y sedes/agencias

10. **Aspectos técnicos destacados**
   - tipado fuerte con TypeScript
   - arquitectura full-stack moderna
   - trazabilidad operativa
   - diseño responsivo mobile-first

11. **Resultados y logros**
   - digitalización del proceso
   - centralización de la información
   - mejora del control operativo
   - soporte para decisiones gerenciales

12. **Limitaciones actuales y trabajo futuro**
   - múltiples técnicos por ticket
   - catálogo formal de sedes/agencias
   - pruebas automatizadas
   - app móvil/offline

13. **Conclusiones**
   - impacto técnico
   - impacto organizacional
   - valor del sistema como solución real

14. **Diapositiva final**
   - “Gracias”
   - espacio para preguntas

### Instrucciones adicionales
- incluye una diapositiva con diagrama de arquitectura
- incluye una diapositiva con flujo del ticket de extremo a extremo
- incluye una diapositiva con tabla o visual de roles
- incluye una diapositiva con ejemplo de reportes
- usa títulos claros y bullets cortos
- evita saturar las diapositivas
- diseña para una exposición oral de 10 a 15 minutos
